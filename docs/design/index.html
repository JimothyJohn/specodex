▮▮▮ DESIGN LAB · 3 DIRECTIONS ▮▮▮

Same field-manual personality, three ways of holding it together.

The current Specodex UI has the right aesthetic — manila palette, Oswald + IBM Plex Mono, deliberately desaturated brass accent — but the execution is rough: 380px rigid sidebars, ellipsis-clipped chips, eight different z-index values, !important patches. Before propagating a fix into the React app, here are three distinct ways the same vocabulary could resolve. Each renders the same mocked drives table so you can compare apples to apples.

▮▮▮ THE DIRECTIONS ▮▮▮
DIRECTION 01

Bauhaus Catalog

Strict 12-column grid, hairline rules, ALL-CAPS section labels, geometric markers (triangle, square, circle) pulled through the brass-and-stamp palette. Graph-paper background the content actually aligns to.

Hairline rules Strict grid Geometric markers
DIRECTION 02

Aicher Manual

Otl Aicher's 1972 Munich aesthetic. Pictogram-led left rail (one SVG icon per product type and per metric), wide gutters, no rules — separation is by negative space. Density via icons, not labels.

Pictograms Wide space No rules
DIRECTION 03

Vignelli Transit

Massimo Vignelli's NYC subway diagrams meet a Knoll catalog. Heavy rule weights, oversized Archivo Black numerals as the visual hierarchy, one bold color, tight tracking on every header. Numerals do the work labels usually do.

Heavy rules Oversized numerals One color
▮▮▮ WHAT STAYS THE SAME ▮▮▮
  • Palette. Paper #E8E2C9, ink #1A1A14, OD #3A2C1C, brass #866C25, stamp red #7A1F1F. Already shared between docs and app.
  • Type families. Oswald (display) + IBM Plex Mono (body). Each direction adds at most one supporting family for a signature element.
  • Dark theme. Espresso variant from the current app carries through. Demos use the light theme only — dark mode behavior is fully derivable.
  • The "no native chrome" rule. Every primitive shown (chips, dropdowns, modals) renders without OS chrome — same constraint that's in CLAUDE.md today.
