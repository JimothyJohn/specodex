▮▮▮ THE BUG ▮▮▮

_coerce_ip_rating's docstring promised "Anything else becomes None" — the function-end return v violated that for lists, tuples, floats. Worse: isinstance(v, int) is True for booleans in Python (since bool ⊂ int), so True / False were emerging as IP ratings of 1 / 0. Real IP ratings are two digits; bools are garbage.

Fix: explicit isinstance(v, bool) → None guard before the int check, and replace the trailing return v with return None. Per CLAUDE.md "Bug fix → regression test FIRST" — the property test fails before the fix and stays after the fix forever. Plus 5 explicit regression cases under TestCoerceIpRating for list, tuple, float, True, False so the specific shapes hypothesis surfaced are pinned even if strategies drift.
