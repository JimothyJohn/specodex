▮▮▮ WHAT CHANGED ▮▮▮
  1. A-28

    js/clear-text-logging — upload.ts:83

    The post-upload info log was interpolating BUCKET, which is sourced from process.env.UPLOAD_BUCKET. CodeQL flags any log of env-derived values at high because env can carry secrets. Fix: drop the bucket name from the log line entirely. The datasheet ID and S3 key remain — both are non-sensitive and already live in the success response body.

  2. A-1, A-2

    py/incomplete-url-substring-sanitization — test_pricing.py:155, :169

    Two test assertions used substring matching to confirm the resolver produced an OEM URL on the right vendor's domain ("orientalmotor.com" in c.url / "shop1.us.mitsubishielectric.com" in c.url). CodeQL flags the pattern because in production code it can be bypassed by a URL like https://evil.com/?orientalmotor.com — the substring is present at an arbitrary position. These were tests, not security checks, but the safe form documents the resolver contract better. Fix: new _host_matches(url, domain) helper that parses the URL and accepts only an exact host match or a true subdomain.
