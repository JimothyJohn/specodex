▮▮▮ THE TWO SURFACES ▮▮▮
  1. A-1

    Results-list footer banner

    Below the pagination on every result list: "Don't see your manufacturer or part? Suggest one →". Quiet by design — small, low-contrast, dashed top border. Opens <FeedbackModal> with defaultCategory='missing_product' and threads the active productType + filter set into the mail body. Catches "I see results but my brand isn't here", which the empty-state CTA from #74 never sees.

  2. A-2

    Detail-modal "Spec wrong?" link

    Right-aligned at the bottom of the spec table inside ProductDetailModal. Opens <FeedbackModal> with defaultCategory='wrong_info' and attaches { manufacturer, part_number } as context.product so the email body reads "Product: Mitsubishi / HG-KR43". Closes the detail modal first to side-step the click-outside conflict.

▮▮▮ WHY CALLBACK-THEN-CLOSE ▮▮▮

The detail modal closes on any click outside its modalRef. The <FeedbackModal> overlay is mounted at the ProductList tree level (sibling of ProductDetailModal, not nested), so clicks inside the feedback overlay would trigger the detail modal's outside-click handler mid-feedback. Two reasonable fixes:

  1. Tag the feedback overlay with data-portaled-popover so the detail modal's click-outside handler ignores clicks inside it (same trick AddToProjectMenu uses).
  2. Close the detail modal first; the parent captures the product reference in state before close, then opens feedback.

Picked option 2. One extra line, no edits to the modal in the parent PR, and the UX is arguably clearer — the user is moving from "viewing a product" to "writing about a product", so a clean modal swap matches the mental shift.

▮▮▮ FREQUENCY CAPS ▮▮▮

Both surfaces are always visible — no per-session caps, no localStorage flags, no dismissal state. They're quiet enough that pestering isn't a risk: the footer is a small dashed-border line at the bottom of the table, and the detail-modal link is a single underlined text button at the end of the spec list. The frequency-cap machinery from the original proposal (Tier C+ nudges) is deliberately out of scope here — it lands when louder surfaces (toast / pulse) ship.
