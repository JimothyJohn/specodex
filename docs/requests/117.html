SCHEMA Phase 2 — motor_mount_pattern backfill CLI

SCHEMA Phase 1 (PR #87) introduced motor_mount_pattern; every motor row written before that PR has the field null. This PR ships the one-shot backfill that derives the canonical pattern from frame_size and writes it back — code-only, no DynamoDB writes from the sprint. --apply is the explicit opt-in; default is --dry-run (prints a summary and exits).

    apply=False is read-only

    No DB writes regardless of how many rows would match. Prints a summary listing matched count + first 10 sample mappings for spot-check.

  2. I-2

    Writes only derivable, currently-null rows

    Walker skips rows where motor_mount_pattern is already set (idempotent), where frame_size is null/empty (nothing to derive from), or where the normalizer returns None (vendor-specific encodings like "60mm" / "60" — pending a per- vendor lookup PR).

  3. I-3

    Second apply pass is a no-op

    matched=0, written=0; already_set count rises by the first pass's writes.
