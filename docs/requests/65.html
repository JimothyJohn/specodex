▮▮▮ WHAT CHANGES ▮▮▮
  1. 5a · SES

    Cognito sender via SES

    Cognito email defaults are unauthenticated and rate-limited. Wires SES as the sender for verification + reset emails, opt-in via env so dev/local stay on the no-ops path. New app/.env.example entries; CDK stack picks them up.

  2. 5c · REVOKE

    Refresh-token revocation on logout

    Logout previously dropped only the access token; the refresh token stayed valid until natural expiry. Backend now calls cognito-idp:RevokeToken on the refresh handle as part of the logout flow. Frontend client mirrors with an explicit authLogout().

  3. 5e · AUDIT

    Structured audit logging on /api/auth/*

    New auth-audit.ts middleware emits a structured CloudWatch line per request — IP, user-agent fingerprint, outcome, latency. 117 lines of middleware, 241 lines of tests pinning the field set. Forensic surface for the next time someone asks "who logged in from where, when."

  4. 5f · ALARMS

    WAF CloudWatch alarms + SNS topic

    WAF was already enabled on the API; alarms were not. Adds a CloudWatch alarm on blocked-request-rate and an SNS topic so an actual page goes out when someone starts probing. Touches the frontend stack (CSP) + alarms imports + body.
