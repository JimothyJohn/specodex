▮▮▮ WHAT CHANGES ▮▮▮
  1. A-1

    Two property tests at 100 examples each

    test_only_known_exception_types_or_well_formed_list: st.binary(0–256) covers broad random bytes. test_pdf_magic_with_garbage_tail_no_unexpected_exceptions: prepends "%PDF-1.4\n" so PyMuPDF surface-recognises the input, then random tail — targets the "PyMuPDF thinks it's a PDF, actually isn't" corner.

  2. A-2

    Contract: success returns list[int], failure raises typed exception

    Success path returns a sorted, non-negative list of 0-indexed page numbers. Failure path raises FileDataError or EmptyFileError (both RuntimeError subclasses), which the caller's try/except can catch. A KeyError / TypeError / AttributeError escaping is the regression to catch.

  3. A-3

    Resolves fitz exception tuple at import time

    If PyMuPDF restructures its hierarchy in a future upgrade, the test fails informatively rather than silently passing under a broader-than-intended except.
