▮▮▮ THE BUG ▮▮▮

if v is falsy for 0, empty string, False, empty containers — falling through to str(val) which produced the literal Python repr of the input dict. {"value": 0, "unit": "mm"} became "{'value': 0, 'unit': 'mm'}" (0 is a real value, e.g. backlash spec). {} became "{}". Both rendered as junk in the frontend's free-text columns AND inflated quality scores (a non-None field counts as populated even if it's gibberish).

Fix: explicit "no meaningful value" check — v is None or whitespace-only string. Format everything else, including 0 and other falsy-but-real values.
