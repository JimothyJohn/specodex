▮▮▮ HOW TO READ THESE ▮▮▮
  1. RL-01

    One page per PR

    Every Specodex PR ships with a static HTML page named by its number. Newest at the top of the log; the page itself stays put once written.

  2. RL-02

    Source is GitHub

    Title, body, branch, merge-date, files come from gh pr view <n>. The page is a render, not a paraphrase — the GitHub PR is still the authoritative record for review threads.

  3. RL-03

    Same field-manual style

    Manila paper, OD-green band, IBM Plex Mono body. Every page in the log shares the palette and grid as the catalog landing page — same shop, different tab.

  4. RL-04

    Link from the band

    The "Pull Requests" tab in the top band of every page (catalog included) brings you back here. Don't bury this — it's the public face of how the project moves.
