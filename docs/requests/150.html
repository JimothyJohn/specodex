▮▮▮ WHAT WAS BROKEN ▮▮▮
  • Filter chip silent truncation. .filter-attribute had white-space: nowrap; overflow: hidden; text-overflow: ellipsis on a chip that was effectively 380 px wide. Long names like network_protocol or encoder_resolution_bits clipped into Re… — invisible information loss. This is the "text is being truncated" complaint that motivated the redesign.
  • Header soft-shadow drift. The header used box-shadow: 0 2px 6px var(--shadow-md) + a 2 px ink border underneath. The shadow blurred the boundary, undermining the field-manual personality. Field manuals have rule lines, not shadows.
  • Token chaos. The audit found 12 different z-index values from 0 to 10000 scattered across App.css. Five bare font-family: monospace declarations bypassing the --font-mono cascade — Plex Mono's tabular figures and the cross-platform fallback chain were being lost at those sites, contributing to "things weren't lining up."
  • Web-pill chip shapes. .filter-chip-minimal used border-radius: 4px + a 135° gradient fill. On a manila paper background that reads as a button from a different design language. Squared cards read as catalog.
▮▮▮ THE FIX ▮▮▮
  • Tokens layer. Both :root[data-theme="light"] and the dark/default :root blocks now declare a 5-tier z-index scale (--z-content / --z-toolbar / --z-header / --z-popover / --z-modal / --z-toast) and two rule tokens (--rule-fine / --rule-strong). Call sites migrate piecewise; the only one this PR migrates is .header itself.
  • Global tabular numerals. body sets font-variant-numeric: tabular-nums. The existing 18 component-level declarations stay, but new numeric surfaces no longer have to remember to opt in.
  • Five font-family: monospace sites rewritten to var(--font-mono). Mechanical, but it brings those surfaces back into the Plex Mono cascade.
  • Header polish. Drop the box-shadow, raise the bottom border 2 px → 3 px ink, use var(--z-header) for the z-index. .nav-btn goes from border-radius: 2px to squared, letter-spacing 0.14em → 0.16em (matches the stencil treatment everywhere else).
  • Filter chip — the truncation fix. .filter-attribute switches to multi-line wrap: word-break: break-word; overflow-wrap: anywhere; min-width: 0. Long names extend the chip by one line; nothing clips silently. Same selector now uses Oswald uppercase + letter-spacing 0.16em — the Bauhaus stencil signature applied where users read filter names most.
  • Filter chip — squared corners. .filter-chip-minimal: border-radius: 4px → 2px, drop the 135° gradient for a flat var(--bg-secondary) fill.
  • Column header stencil. .column-header-label-text takes Oswald uppercase + letter-spacing 0.16em. The prior proportional-cased mono with ellipsis was the second truncation surface the audit called out; the condensed headline fits more text before ellipsis kicks in.
