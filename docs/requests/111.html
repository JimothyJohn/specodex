▮▮▮ THE CONTRACT ▮▮▮

From the docstring: (1) returns a List[<schema_type instance>] when at least one row validates; (2) raises ValueError when the response is unusable (missing .text, empty after fence-strip, not valid JSON, top-level scalar, zero rows validated); (3) never raises any other exception type — a KeyError / TypeError / AttributeError escaping is the regression to catch, since the parser is the isolation barrier between Gemini's text and the rest of the pipeline; (4) never returns half-validated objects — every list element is a real Pydantic instance.

▮▮▮ WHAT CHANGES ▮▮▮
  1. A-1

    RAW_TEXT strategy — 4-way mix

    Pure JSON, fence-wrapped JSON ("```json…```"), plain non-JSON text, and JSON truncated mid- string. test_only_raises_value_error_or_returns_list runs 200 examples against this strategy.

  2. A-2

    Pure-JSON strategy — st.recursive

    Recursive JSON values (primitives at the leaves, lists / dicts at internal nodes). Catches the case where the parser pre-decodes successfully but per-row Pydantic validation misbehaves on a weird input.

  3. A-3

    Context-merge strategy

    Caller-provided context dicts with arbitrary identifier keys + primitive values. Verifies the merge step doesn't propagate weird-input failures past the per-row exception swallowing.

  4. A-4

    5 example-based sanity cases

    Documented ValueError paths: None response, no .text attr, empty text, top-level string, top-level number. Covers the cases hypothesis is unlikely to hit reliably.
