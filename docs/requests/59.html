DB cleanup — gearhead torque rename + electric_cylinder field drops

Driven by a field-coverage audit (cli/audit_fields.py) of products-dev (6,578 PRODUCT rows). The headline finding: the Gearhead model declared rated_torque (0% hits) and peak_torque (0.6%), but the LLM had been emitting max_continuous_torque (51.8%) and max_peak_torque (51.2%) all along. The data was landing in DynamoDB but Pydantic couldn't read it back. This PR aligns the model with reality and ships the audit CLI that found it.

▮▮▮ WHAT CHANGES ▮▮▮
  1. FIX-1

    Gearhead torque rename

    rated_torque → max_continuous_torque and peak_torque → max_peak_torque. gearhead.md ADR already documented the new names — this PR aligns the code with the ADR. Touches gearhead.py, adapters.py port mapping, spec_rules.py bounds, intake column-list comment, query spec list, test fixtures, and the regenerated generated.ts.

  2. DROP

    Two semantically-wrong fields

    ElectricCylinder.gear_ratio (0% hits — rotary-mechanical, doesn't apply to a linear product) and ElectricCylinder.linear_speed_at_rated_load (0% hits — duplicates max_linear_speed) are removed.

  3. CLI

    cli/audit_fields.py — new

    Read-only field-coverage audit. Filters PK to PRODUCT#* so co-tenant INGEST#* ingest-log rows don't contaminate the stats. Output drives todo/DB_CLEANUP.md Phase 2+ (lead_time / warranty / msrp population).

  4. KEEP

    lead_time and warranty NOT dropped

    Per Nick's instruction. They land at 0% on every product type today, but the right move is to populate them in Phase 2, not delete them — these are commercial fields users will eventually filter on.
