  2. P-1

    Python production (12 alerts)

    Mostly small touch-ups: drop two unused module-level loggers (cli/audit_units.py, specodex/models/common.py); pre-bind two locals whose flow analyzers couldn't track (cli/query.py num_filter; specodex/scraper.py validated_api_key); explanatory comments on three empty-except handlers (quickstart.py health-check retry, spec_rules.py non-nullable Pydantic skip, utils.py SECLEVEL fallback); raise SystemExit instead of sys.exit in schemagen._detect_mime so flow analyzers see NoReturn; switch one import x as y to from pkg import x as y to use a single import style across schemagen.py; wrap intentional multi-line strings in parens in audit_dedupes.py so CodeQL sees them as expressions, not list-comma typos.

  3. P-2

    cmd_dev: bare open() → with-block

    Two py/file-not-closed alerts on the backend_log / frontend_log opens in cmd_dev. The existing code closed them in the signal handler, which works in practice but isn't traceable by CodeQL. Refactor to a parenthesized with open(...) as backend_log, open(...) as frontend_log: block — the signal handler's manual close goes away because sys.exit raises SystemExit, which propagates through the with-block and runs __exit__. Slightly larger diff (~70 lines indented one level), zero behavior change.

  4. T-1

    Tests: side-effect-in-assert (8 alerts)

    Across test_admin.py, test_db.py, test_db_integration.py, and test_resilience.py: extract mutation calls (bl.remove(...), client.delete(...), db.delete(...)) to a local before asserting on the result. Why it matters: under python -O asserts strip from the bytecode entirely; if the mutation lives inside the assert expression it vanishes too, so any test that depends on the prior mutation having happened silently fails on the next assertion. Extracting first keeps the side effect.

  5. T-2

    Tests: empty-except (2 alerts)

    test_performance.py:71, 88 — except Exception: pass with no comment. The perf test deliberately measures the failure path (1000 iterations × 4 invalid inputs). One-line comment inside each except block.
