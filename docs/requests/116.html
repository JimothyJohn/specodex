▮▮▮ THE BUG ▮▮▮

Line 44 of drive.py's _coerce_protocol_list read out.append(mapped if mapped else ("unknown" if item.strip() else item)). For empty strings item.strip() is falsy, so the original item (the empty string) passed through. The Pydantic List[EncoderProtocol] validator then rejected the empty string, killing rows whose encoder_feedback_support happened to include an empty entry from a bad upstream parse.

Fix: simplify to mapped if mapped else "unknown". Drops the empty-string-passthrough escape valve, which had no useful behaviour. Per CLAUDE.md "Bug fix → test FIRST" — the property test fails before the fix and stays after it forever.
